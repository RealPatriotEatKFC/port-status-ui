# 네트워크 포트 상태 관리 시스템

서버실에 포트 상황을 항상 엑셀에만 정리하고자 하니 불편해서 만들었습니다.
서버실의 네트워크 장비 포트 상태를 웹 인터페이스로 관리할 수 있는 시스템입니다. 현장에서 직접 확인한 포트 상태를 클릭만으로 쉽게 등록하고 관리할 수 있습니다.
추후 로컬뿐만 아니라 외부에서 접근 후 수정할 수 있게 만들 예정입니다.

## 프로젝트 개요

네트워크 장비의 포트 상태를 시각적으로 관리하기 위한 웹 애플리케이션입니다. 서버실에 직접 방문하여 육안으로 확인한 포트 상태를 직관적인 UI를 통해 입력하고 관리할 수 있습니다.

### 주요 목적
- 네트워크 장비의 포트 상태를 시각적으로 관리
- 현장에서 직접 확인한 정보를 빠르게 입력
- 장비별 포트 레이아웃을 유연하게 설정
- 포트 상태를 색상으로 구분하여 한눈에 파악

## 주요 기능

### 1. 장비 관리
- **장비 추가**: 장비 이름, 위치, 타입, 포트 레이아웃(행/열) 설정
- **장비 목록**: 등록된 모든 장비를 목록으로 표시
- **장비 선택**: 클릭하여 해당 장비의 포트 그리드 확인
- **장비 삭제**: 불필요한 장비 삭제 기능
- **장비 필터링**: 검색어, 위치, 타입으로 장비 필터링

### 2. 포트 상태 관리
- **포트 그리드**: 선택한 장비의 포트를 그리드 형태로 시각화
- **포트 상태 설정**: 각 포트를 클릭하여 상태 입력
  - **정상** (초록색): 정상 작동 중인 포트
  - **미사용** (회색): 사용하지 않는 포트
  - **점검필요** (주황색): 점검이 필요한 포트
- **포트 정보 입력**: 포트 이름, 상태, 메모 입력
- **동적 레이아웃**: 브라우저 크기에 맞춰 포트 크기 자동 조절

### 3. 사용자 인터페이스
- **사이드바 토글**: 좌측 장비 목록을 숨기기/보이기 기능
- **반응형 디자인**: 화면 크기에 따라 자동 조절
- **직관적인 색상 코딩**: 포트 상태를 색상으로 구분
- **부드러운 애니메이션**: 사이드바 토글 및 상태 변경 시 애니메이션

### 4. 데이터 관리
- **로컬 저장**: 브라우저 localStorage에 자동 저장
- **데이터 영구 보존**: 새로고침 후에도 데이터 유지
- **실시간 업데이트**: 상태 변경 시 즉시 반영

## 기술 스택

### Frontend
- **React 19.1.1**: 사용자 인터페이스 구축
- **TypeScript**: 타입 안정성 보장
- **Vite**: 빠른 개발 환경 및 빌드 도구
- **Tailwind CSS**: 유틸리티 기반 스타일링 (3.4.17 버전!!!)
- **Zustand 5.0.8**: 상태 관리 (persist 미들웨어로 localStorage 연동)

### 개발 도구
- **ESLint**: 코드 품질 관리
- **PostCSS**: CSS 처리
- **Autoprefixer**: 브라우저 호환성

## 주요 구현 사항

### 1. 상태 관리 (Zustand)
- **Persist 미들웨어**: localStorage에 자동 저장/복원
- **타입 안전성**: TypeScript로 모든 상태 타입 정의
- **액션 기반 업데이트**: 명확한 액션 함수로 상태 변경

```typescript
// 장비 추가
addEquipment: (name, layout, location?, type?) => void

// 포트 상태 업데이트
updatePortStatus: (equipmentId, portId, status) => void

// 필터링
getFilteredEquipments: () => Equipment[]
```

### 2. 반응형 그리드 레이아웃
- **CSS Grid**: 동적 그리드 생성
- **자동 크기 조절**: 브라우저 크기에 맞춰 포트 크기 자동 조절
- **최소 크기 보장**: 포트가 너무 작아지지 않도록 최소 80px 보장
- **가로 스크롤**: 포트가 많을 경우 자동 스크롤

### 3. 동적 UI 조정
- **사이드바 토글**: 사이드바 열림/닫힘에 따라 포트 그리드 위치 자동 조절
- **애니메이션**: CSS transition으로 부드러운 전환 효과
- **상태별 색상**: 포트 상태에 따라 색상 자동 변경

### 4. 사용자 경험 최적화
- **직관적인 인터페이스**: 클릭만으로 포트 상태 입력
- **시각적 피드백**: 호버 효과 및 선택 상태 표시
- **빠른 검색**: 실시간 필터링
- **모달 기반 입력**: 포트 정보 입력을 위한 모달 UI

## 시작하기

### 설치

```bash
# 의존성 설치
npm install

# 개발 서버 실행
npm run dev

# 프로덕션 빌드
npm run build

# 빌드 미리보기
npm run preview
```

### 사용 방법

1. **장비 추가**
   - 좌측 상단의 "장비 추가" 버튼 클릭
   - 장비 이름, 위치, 타입 입력
   - 포트 레이아웃 설정 (행/열 개수)
   - "추가" 버튼 클릭

2. **포트 상태 입력**
   - 좌측 목록에서 장비 선택
   - 포트 그리드에서 포트 클릭
   - 포트 이름, 상태, 메모 입력
   - "저장" 버튼 클릭

3. **장비 필터링**
   - 검색창에 검색어 입력
   - 위치 또는 타입 드롭다운에서 선택
   - 필터 초기화 버튼으로 리셋

4. **사이드바 토글**
   - 좌측 상단의 "<<" 버튼으로 사이드바 숨기기
   - ">>" 버튼으로 사이드바 다시 보이기

## 주요 UI 컴포넌트

### EquipmentList (장비 목록)
- 장비 목록 표시
- 장비 선택 및 삭제
- 필터 UI 포함
- 장비 추가 모달 트리거

### PortGrid (포트 그리드)
- 선택된 장비의 포트를 그리드로 표시
- 포트 상태별 색상 구분
- 반응형 레이아웃
- 포트 클릭 시 모달 열기

### PortModal (포트 모달)
- 포트 정보 입력/수정
- 상태 선택 (정상/미사용/점검필요)
- 포트 이름 및 메모 입력

### EquipmentFilter (필터)
- 검색어 입력
- 위치/타입 필터
- 필터 초기화
